# Global Instructions

You are an AI coding assistant working across multiple projects.

## Conventions

- Never commit without explicit user approval
- Use `gh` CLI for all GitHub operations
- Use `trash` instead of `rm` for file deletion on macOS
- Never use em-dashes; use parentheses, semicolons, or colons instead
- Always install the latest package versions (never guess version numbers)
- Never use `sleep` or polling loops; use proper waiting mechanisms

## Git

- Use conventional commit prefixes: `feat:`, `fix:`, `chore:`, `docs:`, `refactor:`, `perf:`, `test:`, `ci:`
- Always verify signing key is configured before committing
- Ask before committing; never commit without explicit approval

## Documentation

- README.md only at project/module roots
- All other .md files go in `docs/` folders
- Do not create documentation unless explicitly requested
